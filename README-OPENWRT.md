# Инструкция по настройке WireGuard клиента для AntiZapret

> Полная инструкция по настройке подключения к AntiZapret VPN через WireGuard на роутерах с OpenWRT.

---

## 📋 Требования

- Роутер с прошивкой **OpenWRT 23.05.5+** (или совместимой)
- Полученный конфигурационный файл `antizapret.conf`
- Доступ к веб-интерфейсу LuCI и/или SSH

---

## 🔹 Шаг 1: Отключение DNS Rebinding

1. Откройте веб-интерфейс роутера (LuCI)
2. Перейдите: **Сеть** → **DHCP и DNS** → **Настройки**
3. Снимите галочку с опции `DNS Rebinding` (или добавьте разрешённые домены, если функция работает в режиме белого списка)
4. Нажмите **Сохранить и применить**

---

## 🔹 Шаг 2: Установка пакета WireGuard

1. Подключитесь к роутеру по SSH или откройте **Система** → **Программное обеспечение**

2. Обновите список пакетов:
   ```bash
   opkg update
   ```

3. Установите необходимые пакеты:
   ```bash
   opkg install wireguard-tools kmod-wireguard luci-proto-wireguard
   ```

4. Перезагрузите веб-интерфейс, чтобы появился раздел управления WireGuard

---

## 🔹 Шаг 3: Импорт конфигурации через Web-UI

1. Перейдите: **Сеть** → **Интерфейсы** → вкладка **WireGuard** (или **Создать новый интерфейс** → тип `WireGuard VPN`)

2. Используйте функцию импорта или вставьте содержимое файла `antizapret.conf`, полученного от сервера AntiZapret

3. Сохраните интерфейс (не применяйте изменения до выполнения Шага 4)

---

## 🔹 Шаг 4: Правка файла конфигурации `antizapret.conf`

Откройте конфиг импортированного интерфейса и внесите следующие изменения:

1. **Address**: замените маску `/32` на `/24`
2. **DNS**: укажите публичный DNS-сервер (например, `1.1.1.1` или `8.8.8.8`)
3. **AllowedIPs**: добавьте DNS-серверы вашего интернет-провайдера и IP-адрес WAN-шлюза для корректной маршрутизации

### Пример конфигурации:

```ini
[Interface]
Address = <ваш_IP_из_конфига>/24
DNS = 1.1.1.1, 8.8.8.8
PrivateKey = <ваш_приватный_ключ>

[Peer]
PublicKey = <публичный_ключ_сервера>
AllowedIPs = 10.29.0.0/16, 10.30.0.0/15, <DNS_провайдера>, <WAN_шлюз>
Endpoint = <адрес_сервера>:<порт>
PersistentKeepalive = 25
```

---

## 🔹 Шаг 5: Настройка фаервола и маршрутизации

### Зоны фаервола

1. Перейдите: **Сеть** → **Фаервол**
2. В настройках интерфейса WireGuard выберите зону: `lan` (или создайте новую зону с доступом в `WAN` и разрешённым форвардингом)
3. Включите `Masquerading` (NAT) для этой зоны

### Маршрутизация

1. Перейдите: **Сеть** → **Маршрутизация** (Static Routes)
2. Убедитесь, что трафик подсетей AntiZapret направляется через интерфейс WireGuard

3. Примените настройки и перезапустите сеть:
   ```bash
   /etc/init.d/network restart
   ```

---

## 🔹 Шаг 6: Проверка работы

1. Перейдите в **Состояние** → **Маршруты** и проверьте наличие активных маршрутов через `wg0` (или имя вашего интерфейса)

2. Откройте консоль и выполните проверку подключения к заблокированным ресурсам

3. При необходимости просмотрите логи:
   ```bash
   logread | grep wireguard
   ```

---

## 📎 Примечания

- Инструкция актуальна для **OpenWRT 23.05.5+** и архитектуры `x86_64`, но логика применима к большинству роутеров на OpenWRT
- Перед любыми обновлениями пакетов обязательно выполняйте `opkg update`
- Если на сервере используются нестандартные порты, убедитесь, что они разрешены в правилах фаервола (разрешить входящие/исходящие UDP)
- Источник: [Форум 4PDA - AntiZapret VPN](https://4pda.to/forum/index.php?showtopic=1095869&st=60#entry133105107)
